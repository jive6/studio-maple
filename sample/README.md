# 仁淀川建設（架空）採用サイト ― 制作サンプル

スタジオメイプルの営業用デモサイト。静的HTML5ページ + CSS + JS。

## 構成

| ファイル | 内容 |
|---|---|
| `build.py` | **生成元。編集するのはここだけ。** |
| `dist/index.html` | 採用トップ（ヒーロー／数字／WLB認証／入口カード／代表メッセージ） |
| `dist/work.html` | 仕事内容（1日の流れ／成長ステップ／Q&A） |
| `dist/voices.html` | 社員の声（インタビュー3本・ダミー原稿） |
| `dist/recruit.html` | 募集要項＋応募フォーム（**擬似送信で完了画面まで動く**） |
| `dist/about.html` | **料金・補助金・制作の流れ（＝営業用ランディング）** |
| `dist/style.css` | 共通スタイル（モバイルファースト） |
| `dist/site.js` | 経路計測／バナー高さ調整／擬似送信 |

## ビルド

```bash
python3 build.py     # → dist/ に一式が出力される
```

`dist/` は毎回まるごと作り直されます。**`dist/` の中を直接編集しないこと**（次のビルドで消えます）。

## 商談ごとの「社名差し替えデモ」

`build.py` の冒頭にある2つの辞書を書き換えて、ビルドし直すだけ。

```python
CONFIG = {          # 社名・住所・電話・創業年・従業員数・代表名
    "company": "株式会社◯◯建設",
    "short":   "◯◯建設",
    ...
}
BRAND = {           # コーポレートカラー
    "primary":      "#15839f",   # ここを相手先の色に変えると全ページ反映
    "primary_deep": "#0c5b70",
    "accent":       "#e59d18",
    ...
}
```

`is_demo` を `False` にすると、フッターの免責と画面下のサンプルバナーが消えます（実案件への移行時）。

## 公開手順（GitHub Pages）

1. `dist/` の中身を公開ディレクトリへ（例：`studiomaple.jp/sample/`）
2. チラシ / FAX / メールのQRには経路パラメータを付ける
   - FAX　　 → `/sample/?src=fax`
   - チラシ　→ `/sample/?src=chirashi`
   - メール　→ `/sample/?src=mail`
3. GA4等の計測タグを入れる場合は `build.py` の `head()` に追記
4. OGP画像 `ogp.png`（1200×630）を `dist/` に置く ※未作成

## 経路計測（`site.js`）

`?src=` は `sessionStorage` に保存され、サイト内を回遊しても消えません。

- サイト内リンクに `?src=` が自動で引き継がれる
- **`about.html` からメールが届くと、本文に「参照元：fax」が自動で入る**
  → GA4を入れなくても、届いたメールを数えるだけで経路別の反応率が出る

## ★公開前に必ず直すもの

- [ ] **`build.py` の `SUBSIDY` 辞書**（補助率・上限・制作費・補助額）が
      プレースホルダー（`◯◯◯,◯◯◯` 等）のまま。**交付要綱で確認して実数に差し替える**
- [ ] `CONFIG` の住所・電話番号（現状 `XXXX` の架空表記）
- [ ] `SM` のスタジオメイプル側の連絡先
- [ ] OGP画像 `ogp.png` の作成
- [ ] `about.html` の「含まれるもの／含まれないもの」を、実際の見積り条件に合わせる

## 差し替えポイント（実案件時）

- 写真プレースホルダー（`.photo`）→ 取材写真
- WLB認証バッジ（`.cert` 内のSVG）→ 実際の認証マーク画像
- 応募フォーム → 実送信（`site.js` の擬似送信を外し、フォーム送信先を設定）
- `<meta name="robots" content="noindex, nofollow">` を外す
- JobPosting 構造化データ（Googleしごと検索）を `recruit.html` に追加

## 注意

- 全ページ `noindex` 済み（検索に載せない）
- 架空である旨の免責はフッターに記載。**削除しないこと**
